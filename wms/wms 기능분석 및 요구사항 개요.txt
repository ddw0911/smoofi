## WMS 분석 및 요구사항

-------------------------------------------------------------------------------------

1. 입고(Receiving)

입고 계획 수립: '공급업체'로부터의 '물품'이 '도착할 예정인 날짜와 시간' 등을 계획합니다. 이 계획은 WMS에 입력되어, '필요한 인력과 장비'가 준비되도록 합니다.
->
data 공급업체(이름, 연락처, 담당자, 주소 등), 물품(고유번호(pk), 크기, 무게 등), 도착할 예정인 날짜와 시간, 필요한 인력과 장비(수량, 개인정보, 장비이름)
method 필요자원 준비여부(boolean)

입고 검수: '창고'에 '도착한 물품'을 검수합니다. '물품의 수량, 품질, 포장 상태'를 확인하고, '송장'과 실제 물품을 '비교'합니다.
->
data 창고, 도착한 물품 , 송장
method 물품 검수 여부 및 결과(boolean), 송장과 비교결과(boolean)

데이터 입력: 검수 완료 후, '물품의 정보'(품목, 수량, 위치 등)를 'WMS에 입력'합니다. 바코드나 RFID '태그를 스캔'하여 자동으로 정보를 기록할 수도 있습니다.
->
data 물품정보(품목, 수량, 위치 등)
method 바코드or태그 스캔 -> 물품정보를 수집, wms로 입력

입고 처리: '물품'이 ''지정된 창고 위치'로 이동'되고, '해당 위치를 WMS에 기록'하여 '재고를 업데이트'합니다.
->
data 물품, 창고위치
method 물품이동, 위치를 wms로 기록, 재고 업데이트

-------------------------------------------------------------------------------------

2. 재고 관리(Inventory Management)

재고 추적: WMS는 재고 수준을 실시간으로 추적하고 관리합니다. 재고의 이동, 보관 상태, 유통 기한 등을 모니터링합니다.
->
data 재고정보(고유번호, 위치, 품목, 상태, 주의사항 등)
method 재고 추적 (재고데이터 수집)

재고 조사: 주기적으로 재고 조사를 실시하여 실제 재고와 시스템 상의 재고가 일치하는지 확인합니다. 불일치가 발생할 경우 원인을 파악하고 수정합니다.
->
data 실재고, 전사재고
method 실사확인 + 불일치 발생 시 처리

재고 최적화: WMS는 재고 수준을 분석하여 재고 과잉이나 부족 문제를 예방합니다. 필요한 경우 재고를 조정하거나 재주문을 합니다.
-> 
data 재고(수량, 상태 등)
method 안전재고 설정, 재고회전율 분석

-------------------------------------------------------------------------------------

3. 보관(Storage)
위치 할당: 입고된 물품은 WMS에 따라 적절한 위치에 저장됩니다. 이는 공간 효율성을 극대화하고, 접근성을 고려하여 결정됩니다.

물품 분류: 물품의 특성에 따라 적절한 보관 방법을 적용합니다. 예를 들어, 온도 조절이 필요한 제품은 별도의 구역에 저장합니다.

위치 관리: WMS는 물품의 정확한 위치를 추적하고, 피킹 작업 시 적절한 위치를 안내합니다.

-------------------------------------------------------------------------------------

4. 출고(Picking and Packing)
주문 수집: 고객의 주문이 들어오면 WMS는 주문에 필요한 물품을 확인하고, 피킹 작업을 지시합니다.

피킹(Picking): 작업자는 WMS의 지시에 따라 창고에서 필요한 물품을 선택하여 수거합니다. WMS는 물품의 위치와 피킹 순서를 최적화하여 작업 효율을 높입니다.
-> 
data 수거할 물품정보(고유번호, 위치)
method 수거 순서?

포장(Packing): 피킹된 물품은 포장 작업이 진행됩니다. WMS는 포장 지침과 라벨링 요구 사항을 제공합니다.
-> 
data 피킹된 물품정보
method 포장(라벨링)지침 

출고 준비: 포장 완료 후, 물품은 출고 준비가 완료된 상태로 지정된 출고 구역에 배치됩니다. WMS는 출고 서류와 송장을 생성합니다.

-------------------------------------------------------------------------------------

5. 출고(Shipping)

출고 검수: 출고할 물품을 최종적으로 검수하고, 정확한 품목과 수량이 출고될 준비가 되었는지 확인합니다.
-> 
data 출고할 물품정보
method 출고검수여부 + 통과여부

운송 관리: 운송 수단에 물품을 적재하고, 운송 일정을 관리합니다. WMS는 운송 상태를 추적하여 실시간으로 업데이트합니다.
->
data 운송수단정보, 운송정보(일정, 상태 등)
method 물품적재, 운송상태 업데이트

배송 정보 제공: 고객에게 배송 정보와 추적 번호를 제공하여 주문 상태를 확인할 수 있도록 합니다.
->
data 배송정보(송장번호, 운송상태), 추적번호
method 배송 및 추적번호제공 

-------------------------------------------------------------------------------------

6. 리포트 및 분석(Reporting and Analytics)
데이터 수집: WMS는 입고, 출고, 재고 등 창고 운영의 모든 데이터를 수집합니다.

보고서 생성: 수집된 데이터를 기반으로 다양한 리포트를 생성하여 운영 상태를 분석합니다. 예를 들어, 재고 회전율, 피킹 효율성, 입출고 트렌드 등을 분석합니다.

의사결정 지원: 리포트와 분석 결과를 활용하여 경영진이 더 나은 결정을 내릴 수 있도록 지원합니다. 재고 관리 전략, 인력 배치, 주문 처리 개선 등 다양한 분야에서 활용됩니다.
