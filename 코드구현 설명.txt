<창고관리 코드설명>
1. 창고등록
0) (선택사항) 창고ID는 등록날짜, 지역 등을 조합해 ID로 연상이 가능하도록 naming
1) 창고수용량은 숫자만 입력되도록 (그림참조)
2) 현재가용량 데이터는 입력받지 않지만 등록 시 수용량과 같은값으로 입력되도록 설정
(추후 섹션이 등록, 수정되면 섹션수용량만큼 자동으로 창고가용량이 차감되도록 trigger)

2. 창고조회
각 속성별로 데이터 출력 (비고 null가능)

3. 창고가용량 조회
1) 등록된 창고리스트와 데이터 출력
2) 창고ID를 입력받아 가용량만 출력 (신규 섹션 등록이 필요할 때 쓰려고 만든기능)

4. 창고지역별 조회
창고소재지를 입력받아 해당지역의 창고데이터만 출력

5. 창고수정 및 삭제
1) 창고ID로 해당 창고데이터 조회(확인 후 수정 및 삭제하도록)
2) 수정or삭제 선택
3) 수정 시 원하는 항목번호 입력 -> 수정할 내용 입력 -> loop: 0을 입력받아 입력 종료

6. 섹션등록
1) 섹션을 등록할 창고ID 입력받
2) 등록가능한지 필요데이터 먼저 입력(가로 세로 높이) 
3) 입력한값을 자동계산히야 현재가용량보다 큰지 계산하여 등록가능여부 출력 like 등록가능
4) 등록가능 시 ID와 타입 설정
5) 등록완료

7. 섹션타입별 조회
1) 섹션타입 입력
2) 해당타입의 섹션데이터만 조회가능

<출고관리 코드설명>
1. 출고요청 등록 by 일반회원
1) 일반회원이 출고탭으로 이동 -> 나의 출고 메뉴 1. 출고요청 2. 출고예정현황
2) 출고요청 선택 시 본인의 출고요청현황 데이터 출력 후 출고요청관련 메뉴
3) 출고요청 등록 시 필수표시사항(*) 적용해가며 입력 (주문요청사항, 비고 null가능)

2. 나의 현황 by 일반회원
1) 본인관련 출고데이터(출고요청ID, 출고물품번호, 주문수량, 출고여부 등)

3. 출고요청 수정요청 by 일반회원
1) 출고요청ID를 입력받음 -> 출력된 ID중에서 선택하여 입력
2) 수정(1)할지 삭제(2)할지 메뉴선택 (수정선택 시 수정내용도 입력)

4. 수정요청 조회 by 관리자
1) 확인할 요청(수정요청or삭제요청)을 입력하면 해당 요청들만 출력

5. 주문자별 출고요청조회 by 관리자
1) 주문회원ID를 입력받아 해당회원의 출고요청만 조회

6. 출고검수 조회
0) 검수는 일반회원이 출고등록하면 설정한 초기데이터로 자동생성
1) 조회시 ID, 연계된 출고요청ID, 검수결과, 검수자, 검수시간, 비고 확인가능

7. 처리상태별 출고요청조회
1) 메뉴에서 선택해 해당 처리상태인 검수데이터의 ID값만 출력
